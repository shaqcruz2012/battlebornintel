## Edge Data Structure

All edges follow this standardized format:

```javascript
{
  source: "entity_prefix_id",      // i_vendor_id, x_company_id, u_university_id, v_fund_id, f_fund_id
  target: "c_100-105",              // Company IDs 100-105
  rel: "invested_in|partners_with|funded_by|filed_with|acquired_by|etc",
  note: "Detailed description of relationship",
  investment: 1000000,              // Optional: Investment amount in USD
  y: 2024,                          // Year of relationship
  stage: "Seed|Series A|Series B",  // Optional: Funding stage
  lead: true|false,                 // Optional: Lead investor indicator
  program: "Program Name"            // Optional: Grant/SBIR program name
}
```

## Entity Prefixes Used

- `i_` - Venture Capital/Angel Investors
- `x_` - External corporations, government agencies, universities
- `f_` - Funds (Battle Born Venture, FundNV, etc.)
- `v_` - Risk capital sources (SBIR, NIH, SBA)
- `c_` - Company IDs
- `u_` - University departments/schools

## Integration Notes

1. **Investor Entity IDs** need to be created with standardized prefixes:
   - Venture firms: `i_sequoia`, `i_greylock`, `i_boldstart`, etc.
   - Angel networks: `i_angel_fintech`, `i_techcrunch`
   - Corporate VCs: Prefix with company, e.g., `x_stripe_v` for Stripe Ventures

2. **Corporate Partner IDs** use `x_` prefix:
   - Fortune 500: `x_salesforce`, `x_google`, `x_stripe`
   - Sector leaders: `x_databricks`, `x_snowflake`, `x_workiva`

3. **University IDs** use `u_` prefix with school/department:
   - `u_stanford_cs`, `u_berkeley_eecs`, `u_mit_csail`

4. **Risk Capital Sources** use `v_` prefix:
   - `v_sbir` - Small Business Innovation Research
   - `v_nih` - National Institutes of Health
   - `v_sba` - Small Business Administration

5. **Battle Born Venture** connections use `f_bbv` as source for each investment
